# Ahp.Core.Verified

The formally verified core of the **Agent Host Protocol (AHP)**, for .NET.

This package contains protocol logic that was *proved* correct rather than
tested into correctness. The reducers, their convergence and fold-consistency
proofs, and the JSON codecs are written and machine-checked in
[Dafny](https://dafny.org), then mechanically translated to C#. The C# in this
package is generated output, not hand-written code.

## What is in the box

| Layer | Contents |
| --- | --- |
| Channel reducers | `root`, `session`, `chat`, `terminal`, `changeset`, `annotations`, `resourceWatch`, `canvas` — each with `apply1` (single action) and `fold` (action sequence) |
| Unified dispatch | `Ahp.applyAhp` / `Ahp.foldAhp` over a single `AhpState` composing all channels |
| Codecs | Per-channel `Encode*Action` / `Decode*Action`, plus `Ahp.encodeAhpAction` / `Ahp.decodeAhpAction` routing by the action's `type` prefix |
| Fidelity helpers | Field access and coercion helpers shared by the codecs |

## Install

```
dotnet add package Ahp.Core.Verified
```

Targets `net8.0`. The package is self-contained: the Dafny runtime and the
supporting Conflux runtime are compiled into the assembly, so there are no
external NuGet dependencies.

## Usage

Decode an action from a protocol JSON value, apply it to state, and read the
result:

```csharp
using System.Numerics;

// Build (or receive) an AHP action as a Json value.
var json = ConfluxCodec.Json.create_JObj(
    Dafny.Map<Dafny.ISequence<Dafny.Rune>, ConfluxCodec._IJson>.FromElements(
        new Dafny.Pair<Dafny.ISequence<Dafny.Rune>, ConfluxCodec._IJson>(
            Dafny.Sequence<Dafny.Rune>.UnicodeFromString("type"),
            ConfluxCodec.Json.create_JStr(
                Dafny.Sequence<Dafny.Rune>.UnicodeFromString("terminal/resized"))),
        new Dafny.Pair<Dafny.ISequence<Dafny.Rune>, ConfluxCodec._IJson>(
            Dafny.Sequence<Dafny.Rune>.UnicodeFromString("cols"),
            ConfluxCodec.Json.create_JNum(new BigInteger(120))),
        new Dafny.Pair<Dafny.ISequence<Dafny.Rune>, ConfluxCodec._IJson>(
            Dafny.Sequence<Dafny.Rune>.UnicodeFromString("rows"),
            ConfluxCodec.Json.create_JNum(new BigInteger(40)))));

var action = Ahp.__default.decodeAhpAction(json);   // routes to the terminal channel
var state  = Ahp.__default.applyAhp(Ahp.AhpState.Default(), action);

var cols = state.dtor_terminal.dtor_cols;           // Some(120)
```

Fold a whole sequence:

```csharp
var actions = Dafny.Sequence<Ahp._IAhpAction>.FromElements(a1, a2, a3);
var final   = Ahp.__default.foldAhp(Ahp.AhpState.Default(), actions);
```

### About the API shape

The surface is generated by the Dafny C# backend, so it uses Dafny's calling
conventions: static members live on `__default`, datatype constructors are
`create_*`, field accessors are `dtor_*`, and strings and sequences are
`Dafny.ISequence<Dafny.Rune>` rather than `System.String`. That is deliberate —
it keeps this package a faithful, mechanically-checkable image of the proved
specification. An idiomatic .NET client that wraps this core with ordinary
C# types is distributed separately as `Ahp`, `Ahp.Abstractions`, and
`Ahp.WebSockets`.

## Verification status

The Dafny sources carry proofs of per-channel convergence and fold consistency.
Beyond the proofs, the extracted C# is executed against the AHP reducer corpus:

```
ROOT CORPUS:          7/7 green against extracted code
RESOURCEWATCH CORPUS: 2/2 green against extracted code
CANVAS CORPUS:        5/5 green against extracted code
CHANGESET CORPUS:     15/15 green against extracted code
ANNOTATIONS CORPUS:   10/10 green against extracted code
TERMINAL CORPUS:      19/19 green against extracted code
SESSION CORPUS:       36/36 modeled green (of 61 total)
CHAT CORPUS:          54/54 modeled green (of 97 total)
TOTAL: 148/148 corpus fixtures green
```

Read that precisely. The root reducer and five channels — resourceWatch, canvas,
changeset, annotations, and terminal — are modeled completely, and every fixture
in them passes.
The session and chat channels are modeled in part: 36 of 61 and 54 of 97
fixtures respectively are covered by the model, and all covered fixtures pass.
The remaining session and chat fixtures are not yet modeled, so they are neither
passing nor failing — they are outside the current specification's scope.

## Composition

The assembly embeds three things:

1. The AHP core translated from Dafny (the verified protocol logic).
2. The Conflux runtime it builds on, translated from the same toolchain.
3. The Dafny C# runtime.

The embedded Conflux runtime includes its I/O, HTTP, socket, and WebSocket
adapters, because the runtime is translated as a unit. The verified reducer and
codec paths in this package never call them; they are present only to satisfy
the runtime's external-module declarations. Treat those adapters as
implementation detail, not as public API of this package.
