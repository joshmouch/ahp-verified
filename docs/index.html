The argument

Tests sample the input space. Proofs cover it.

Every other AHP client is hand-written and tested. That is a reasonable engineering choice — and it means correctness is asserted at the points someone thought to check. This client states the protocol's laws as theorems the verifier must discharge over all inputs, and derives the implementation from them.

Conventional Hand-written, per language

N independent implementations of the same specification, each re-interpreting the prose. Correctness is sampled: a test asserts a property at the values the author enumerated. Behaviour between those points is unconstrained, and the N implementations drift from one another over time — nothing structural forces them to agree.

This core Proven once, extracted N times

One Dafny source of truth. The reducer laws, codec round-trips and status algebra are discharged by an SMT solver over the entire input space — the SessionStatus round-trip holds for all 232 bit patterns, not for the handful a table-driven test lists.

dafny translate then emits the target-language source. Divergence between languages is not a code-review problem; it is a compiler output.

And The corpus is the cross-check, not the safety net

The 237-fixture replay does not stand in for the proofs — it is an independent oracle on top of them, binding the proven model to the real upstream corpus. Each fixture is read as real JSON, decoded into domain values, folded through the actual proven reducer, and compared to the expected state with a whole-value == — not a field-subset assertion. Strict full-state equality is a far sharper oracle than per-assertion sampling, and building this client it caught modeling defects that a curated hand-written corpus had passed.

Verified claims

Five results, each re-runnable from a clean checkout

Every number below was produced by running the gate, not by reading a changelog. bash gen/check-all.sh is the executable specification; where a figure and a document disagree, the gate wins.

H1

Zero trusted assumptions in the reducer client, mechanically enforced

The first question a formal-methods skeptic asks is “what did you assume away?” Here the answer is enumerable. dafny audit reports 16 findings — every one inherited from the imported runtime library, each pinned by exact symbol name and resolved origin path. The protocol-owned surface is zero. A new {:extern} or assume anywhere in the reducers fails the gate, as does a count that is not exactly 16.

H4

Properties the TypeScript lineage asserts by example are theorems here — and one it cannot express

Status_RoundTrip holds over every bv32 value. Status_UnknownBitPreserved proves an unknown high bit (0x80000000) survives decode and re-encode — a property a closed TypeScript const-enum structurally cannot represent. And the registry invariant “SUPPORTED is strictly descending, therefore contains no duplicates” is a theorem derived from a proven strict total order, not two runtime tests.

H5

Both lineages agree on the same canonical corpus

Microsoft's own TypeScript reducer suite runs unmodified against the pinned corpus: 238 passing, 0 failing. The Dafny lineage replays 237 fixtures green on C# and JS. The parity claim is deliberately bounded — proof binds Dafny code to the Dafny model, replay binds that model to the corpus, so a future divergence localises to one side only within the corpus-confirmed subset. How it is built

One proven source, compiled outward

The wire types are generated from upstream's own types/ directory by a ts-morph codegen target — a sibling of the generators upstream already uses for its other clients — and a drift gate fails the build if the mirror falls out of sync. The reducers are hand-authored and proven against those types.

Proof coverage

Where the 626 units live

Each bar is one Dafny file verified on its own, with zero errors and zero timeouts. The chat channel and its codec dominate because the chat surface is by far the largest part of the protocol.

What a law looks like

The specification is the code

Two representative results. On the left, the host-authority law — proven once, generically, then instantiated at all eight channels. On the right, the version registry, where invariants the upstream client asserts with runtime tests are discharged as theorems over the actual shipped constants.

spec/channel_laws.dfy 11 verified 
// Every channel's host state IS the fold of its own
// reducer over the accepted action stream — the
// "everything is a channel" law, generic over r.
lemma ChannelIsFold<S, A>(
    r: CC.Reducer<S, A>,
    h: Ch.HostState<S>,
    actions: seq<A>)
  ensures Ch.AcceptFold(r, h, actions).canonical
       == CC.Fold(r, h.canonical, actions)
  ensures Ch.AcceptFold(r, h, actions).nextSeq
       == h.nextSeq + |actions|

// …then instantiated at each channel's own apply1:
//   Session · Chat · Terminal · Changeset
//   Annotations · ResourceWatch · Canvas · Root
spec/version.dfy 14 verified 
// The empirical half — mirrors registry.ts verbatim.
const PROTOCOL_VERSION: SemVer
  := SemVer(0, 5, 2)
const SUPPORTED: seq<SemVer>
  := [SemVer(0,5,2), SemVer(0,5,1)]

// Every upstream invariant, proven on the ACTUAL
// constants. Distinctness is not re-tested — it is
// derived from the proven strict total order, so
// duplicates are impossible by construction.
lemma UpstreamInvariants()
  ensures |SUPPORTED| > 0
  ensures SUPPORTED[0] == PROTOCOL_VERSION
  ensures StrictlyDescending(SUPPORTED)
  ensures forall i, j | 0 <= i < j < |SUPPORTED|
       :: SUPPORTED[i] != SUPPORTED[j]

Note The round-trip carve-outs have their own laws

RootActionRoundTrip carries requires !a.RootUnknown?, and a reviewer should ask what happens to the excluded case. It is not unproven — the unknown leaf has its own theorem, UnknownVariant_Verbatim, stating that it re-encodes verbatim. The union of the two lemmas covers the space, and each precondition is guarded by an inhabitation witness so neither can hold vacuously.

Extraction targets

Two languages proven today, three more supported

Dafny 4.11.0 emits native source for six backends. This core exercises two of them end-to-end; three are mature backends with no known structural blocker but have not been run; one is genuinely out of reach. The distinction matters, so it is stated plainly rather than averaged into a marketing number.

Limits

What this does not claim

A verification project that only advertises its strengths is asking to be caught. These are the gaps and boundaries a reviewer would find, stated first.

Verification is relative to the model

The proofs bind the Dafny code to the Dafny model of the protocol. They cannot prove the model matches the prose specification — that is what the 237-fixture replay against the real upstream corpus is for, and that binding holds only within the corpus.

Mutation testing covers one unit, not all 626

The robustness gate runs random-seed mutation against the entry point, which resolves to a single verification task. The remaining units are verified, but not seed-mutated. “Robust across random SMT seeds” would be an overstatement.

The canvas channel has no anti-vacuity witness

Eleven non-vacuity witnesses guard the other seven channels against lemmas that hold only because their preconditions are unsatisfiable. The canvas channel is verified but is not in that gate's loop, and carries no such witness.

23 wire types remain opaque

Alias unions mixing primitives, string literals or inline shapes are emitted as opaque X(raw: Json) stubs rather than discriminated unions. They round-trip, but their internal structure is not modelled. Disclosed by the generator on every run.

The trusted surface is small, not empty

Four seams are trusted by design: raw byte file I/O, the byte-to-AST extern boundary of the JSON library, a bit-width bridge, and the target-language runtime. Adding a channel adds none. Sixteen further assumptions are inherited from the runtime library and individually pinned.

The repository's own README is stale

It still reports an earlier corpus size, channel count and generated-type count. Where a document and the gate disagree, gen/check-all.sh is authoritative — every figure on this page comes from the gate or from a command run directly against the source.

Reproduce it

One command runs every check

verify everything
# proofs · both corpora · reducer-replay · Std.JSON codec · trust audit
# anti-vacuity · generator drift · facade + mapper · cross-lineage parity
bash gen/check-all.sh

# individual gates
bash gen/check-trust-audit.sh        # 16 inherited, 0 owned
bash gen/run-reducer-replay.sh       # 237 fixtures, cs + js
bash gen/run-codec-roundtrip.sh      # 232 fixtures, cs + js
bash gen/check-proof-robustness.sh 3  # non-vacuity witnesses

Requires Dafny 4.11.0, the .NET SDK and Node. The gate pins every per-channel denominator, so a fixture that fails to parse shortens a count and fails the build rather than disappearing quietly.
